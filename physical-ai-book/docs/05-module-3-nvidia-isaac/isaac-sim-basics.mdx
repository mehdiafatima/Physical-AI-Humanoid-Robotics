# Isaac Sim Basics

**NVIDIA Isaac Sim** is a powerful, physically accurate, and photorealistic robotics simulation platform built on NVIDIA Omniverse. It provides a robust environment for developing, testing, and training AI-powered robots, drastically reducing the need for costly and time-consuming real-world experiments. This chapter guides you through the fundamental concepts and practical usage of Isaac Sim.

## 4.1. Isaac Sim Installation and Setup

Isaac Sim is part of the NVIDIA Omniverse platform. Ensure you have a compatible NVIDIA GPU (RTX series recommended) and the necessary drivers installed.

### Installation Steps:

1.  **Install NVIDIA Omniverse Launcher**: Download and install the Omniverse Launcher from the [NVIDIA website](https://www.nvidia.com/omniverse/download/launcher/).
2.  **Install Isaac Sim**: Within the Omniverse Launcher, navigate to the "Exchange" tab, search for "Isaac Sim," and install it.
3.  **Launch Isaac Sim**: Once installed, launch Isaac Sim from the Omniverse Launcher. This will open the Omniverse Kit environment with the Isaac Sim extension loaded.

### Initial Configuration:

-   Ensure your workstation meets the minimum system requirements.
-   Familiarize yourself with the Omniverse Kit interface, including the viewport, USD (Universal Scene Description) stage, and various panels.

## 4.2. Understanding USD: The Universal Scene Description

At the heart of Isaac Sim and Omniverse is **USD (Universal Scene Description)**, an open-source 3D scene description technology developed by Pixar. USD provides a rich and robust way to compose, assemble, and interchange 3D data.

### Key Concepts of USD:

-   **Composition**: Layers of USD files can be composed together to form a final scene, allowing for non-destructive workflows.
-   **Referencing**: Reference external USD assets into your scene.
-   **Variants**: Define different variations of an asset (e.g., different colors, configurations) within a single USD file.
-   **Prims**: The fundamental building blocks of a USD stage, representing objects, materials, lights, etc.

## 4.3. Importing Robotic Assets

Isaac Sim provides a library of pre-built robotic assets. You can also import your own URDF or USD models.

### Importing URDF Robots:

1.  **Open the Import Window**: In Isaac Sim, go to `File -> Import -> URDF`.
2.  **Select URDF File**: Browse to your URDF file (e.g., `my_ros2_package/urdf/two_link_arm.urdf`).
3.  **Configure Import Options**: Adjust parameters like "Fix Base Link," "Merge Fixed Joints," and "Create Physics Scene" as needed.
4.  **Import**: Click "Import" to spawn your robot into the scene.

### Programmatic Import (Python Snippet):

You can also import URDF models programmatically using the Isaac Sim Python API.

```python
from omni.isaac.urdf import _urdf
from omni.isaac.core.utils.nucleus import get_assets_root_path
import omni.isaac.core.utils.stage as stage_utils
import carb

# Get assets root path
assets_root_path = get_assets_root_path()
if assets_root_path is None:
    carb.log_error("Could not find Isaac Sim assets root path!")

# Path to your URDF (assuming it's relative to assets_root_path or a full path)
urdf_path = "omniverse://localhost/Library/Robots/Franka/franka_alt_fingers.usd" # Example Franka
# Or a local URDF: urdf_path = "/home/user/ros2_ws/src/my_ros2_package/urdf/two_link_arm.urdf"

# Get a handle to the URDF importer
urdf_importer = _urdf.acquire_urdf_interface()

# Import the URDF
result, prim = urdf_importer.parse_urdf(
    urdf_path=urdf_path,
    prim_path="/World/Franka",
    import_config=_urdf.ImportConfig(),
)

if result:
    print(f"Successfully imported URDF to {prim.GetPath()}")
else:
    print(f"Failed to import URDF from {urdf_path}")
```

## 4.4. Creating Simulation Scenarios

A simulation scenario involves defining the environment, placing objects, and configuring the physics.

### Essential Elements of a Scenario:

-   **Stage**: The overall scene where your simulation takes place.
-   **Ground Plane**: A floor for objects to rest on.
-   **Lighting**: For realistic rendering.
-   **Props/Obstacles**: Other objects in the environment.
-   **Physics Scene**: Configures gravity, solver type, and other physics parameters.

### Basic Python Scenario (Conceptual):

```python
from omni.isaac.core import World
from omni.isaac.core.objects import DynamicCuboid

# Initialize the Isaac Sim World
world = World(stage_units_in_meters=1.0)
world.scene.add_default_ground_plane()

# Add a robot (assuming it's already imported or created)
# ... robot_prim = world.scene.get_object("YourRobotName")

# Add a dynamic cuboid
cuboid = world.scene.add(
    DynamicCuboid(
        prim_path="/World/random_cuboid",
        name="random_cuboid",
        position=[0.0, 0.0, 0.5],
        scale=[0.1, 0.1, 0.1],
        color=[0.0, 0.0, 1.0]
    )
)

world.reset() # Reset the simulation
print("Simulation scene prepared!")
```

## 4.5. ROS 2 Integration with Isaac Sim

Isaac Sim offers robust integration with ROS 2, allowing you to control simulated robots using ROS 2 nodes and exchange data seamlessly.

### Key Integration Points:

-   **`omni.isaac.ros2_bridge`**: The core extension that bridges Isaac Sim to ROS 2.
-   **ROS 2 Publishers/Subscribers**: Control simulated robot joints, publish sensor data (camera, lidar, IMU), and get ground truth information.
-   **ROS 2 Actions/Services**: Implement complex tasks (e.g., navigation goals, manipulation services) in simulation.
